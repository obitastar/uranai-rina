# 姓名判断 鑑定データ集

御福まつかわや 店頭診断アプリ用の姓名判断データ。Claude Code でこのデータを参照して Web アプリを構築します。

## 流派について

最も普及している **熊崎健翁式（くまさきけんおう式）の81数理** を採用しています。複数の公開ソースを照合してデータを確定しました。

## 構成

### `suri.ts` — 81数理データ
姓名判断の核心。1〜81画それぞれの吉凶・キーワード・解説。

- `SURI[n]` … n画の `{ kikkyo, keywords, summary, detail, saijo?, disputed? }`
- `kikkyo` … 大吉 / 吉 / 吉凶混合 / 凶 / 大凶 の5段階
- `saijo` … 最上吉数（三大吉数15,32,48・六大吉数+16,31,45・七大吉数+33）
- `disputed` … **流派により吉凶判定が分かれる数**（27,30,36,40,46等18件）。店頭で使う際は吉寄りに見せる等の方針を決めておくとよい
- `normalizeKaku(total)` … 82画以上を1〜81に正規化
- `getSuri(total)` … 合計画数から数理データを取得

### `gokaku.ts` — 五格の計算と意味
- `calcGokaku(姓の画数配列, 名の画数配列)` … 天格・人格・地格・外格・総格を算出
  - 霊数処理（1字姓・1字名に補う1）と特殊ケース対応済み
- `GOKAKU_MEANING` … 五格それぞれの意味と重要度（人格＞総格＞地格＞外格＞天格）
- `kakuToGogyo(画数)` … 画数の下一桁から五行を導く
- `judgeSansai(天,人,地)` … 三才配置（天人地の五行関係）の吉凶判定

## ⚠️ 最重要の注意：画数の数え方

**このデータは「画数が分かっている前提」で吉凶を判定します。漢字一字ずつの画数を出す部分は別途必要です。**

熊崎式には独自のルールがあります：

1. **旧字体（正字）で数える** … 沢→澤(16画)、桜→櫻(21画) など。新字体と画数が異なる字がある
2. **部首の特殊な数え方** … さんずい「氵」は水として4画、くさかんむり「艹」は6画、りっしんべん「忄」は4画 など、見た目の画数と異なる扱いをする部首がある
3. **数字・かな** … 一二三…や、ひらがな・カタカナにも固有の画数定義がある

→ **対応方針**：漢字→画数の変換は、専用の画数辞書データか既存ライブラリを用意し、その出力をこの `suri.ts` / `gokaku.ts` に渡す設計にします。店頭用途で精度を求めるなら、熊崎式準拠の画数辞書の整備が次のステップになります。

## 使い方（鑑定の組み立て）

1. 入力された姓名の各文字を、画数辞書で画数に変換
2. `calcGokaku()` で五格を算出
3. 各格の画数を `getSuri()` に渡し、吉凶・解説を取得
4. `GOKAKU_MEANING` の重要度順（人格→総格→…）に結果を提示
5. `kakuToGogyo()` ＋ `judgeSansai()` で三才配置の総合運を補足

## 店頭体験での見せ方（推奨）

専門的にしすぎず、お客さんが楽しめる形に：
**人格（あなたの本質）→ 総格（生涯運）→ 地格（若年・恋愛）→ 三才の総合バランス**

凶数が出ても不安を与えすぎないよう、`detail` は「補い方・活かし方」を添える文体で書いてあります。

## 検証

`npx tsx validate.ts` で全データの網羅性・計算ロジック・三才配置を確認できます。

## データ拡張（Claude Code への指示例）

> 「`suri.ts` の各数に『恋愛運』フィールドを追加して。既存の detail と同じ文体・方針（凶数も活かし方を添える）で、熊崎式の一般的解釈に基づきオリジナルに書くこと」
